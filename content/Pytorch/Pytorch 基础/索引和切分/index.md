# 索引
PyTorch 支持两种语法来访问多维数据，它们在功能上是完全等价的，但在性能和可读性上有区别。

| 特性 | **链式索引** (Chained Indexing) | 逗号分隔索引** (Comma-separated Indexing) |
| :--- | :--- | :--- |
| **语法示例** | `x[0][1][2]` | `x[0, 1, 2]` |
| **执行逻辑** | **分步执行**：<br>1. `x[0]` $\to$ 创建临时 3D 视图<br>2. `[1]` $\to$ 在临时视图上创建 2D 视图<br>3. `[2]` $\to$ 在临时视图上获取最终值 | **一步到位**：<br>一次性解析所有维度索引，直接计算内存偏移量，直接返回最终数据。 |
| **中间对象** | **有**<br>每一步都会生成临时的中间张量对象（视图）。 | **无**<br>不创建任何中间临时张量。 |
| **性能表现** |  **低效**<br>多次函数调用开销 + 临时对象创建/销毁的内存与 CPU 成本。 |  **高效**<br>单次操作，无额外内存分配，CPU 调度开销最小。 |
| **代码可读性** |  **一般**<br>维度多时代码冗长，像“剥洋葱”，意图不够直观。 |  **优秀**<br>清晰表达“在第 $i$ 维取 $a$，第 $j$ 维取 $b$”的并行意图。 |
| **高级功能支持** |  **受限**<br>难以优雅地配合切片 (`:`)、省略号 (`...`) 或布尔掩码混合使用。 |  **完美支持**<br>原生支持切片、省略号、布尔掩码及混合索引（如 `x[0, :, mask]`）。 |
| **官方推荐度** |  **不推荐**<br>仅适用于极简场景或调试，生产环境应避免。 |  **强烈推荐**<br>PyTorch 和 NumPy 的标准最佳实践。 |

```
import torch

# 创建一个形状为 [4, 32, 32, 3] 的 4D 随机张量
x = torch.randn(4, 32, 32, 3)

# 方式一：链式索引 (Chained Indexing)
# 1. x[0]      -> 取出第1张图片，返回形状 [32, 32, 3] 的 3D 张量 (临时对象)
# 2. [1]       -> 在上一步结果中取第2行，返回形状 [32, 3] 的 2D 张量 (临时对象)
# 3. [2]       -> 在上一步结果中取第3列，返回形状 [3] 的 1D 张量
print(x[0][1][2])

# 方式二：逗号分隔索引 (Comma-separated Indexing)
# PyTorch 一次性解析所有维度的索引，直接计算内存地址获取数据
print(x[0, 1, 2])
```


# 切片
切片是提取张量子集的标准方式，语法格式为：
```
[start : end : step]
```
| 参数 | 含义 | 默认值 | 关键规则 |
| :--- | :--- | :--- | :--- |
| **start** | 起始索引 | `0` | 包含该位置 |
| **end** | 结束索引 | 维度大小 | 不包含该位置 (左闭右开 `[start, end)`) |
| **step** | 步长 (采样间隔) | `1` | 正数表示顺向，不支持负数步长 |

基本的使用规则：

| 切片方式 | 意义 |
| :--- | :--- |
| `start:end:step` | 从 `start` 开始读取到 `end`(不包含 `end`)，步长为 `step` |
| `start:end` | 从 `start` 开始读取到 `end`(不包含 `end`)，步长为 1 |
| `start:` | 从 `start` 开始读取完后续所有元素，步长为 1 |
| `start::step` | 从 `start` 开始读取完后续所有元素，步长为 `step` |
| `:end:step` | 从 0 开始读取到 `end`(不包含 `end`)，步长为 `step` |
| `:end` | 从 0 开始读取到 `end`(不包含 `end`)，步长为 1 |
| `::step` | 步长为 `step` 采样 |
| `::` | 读取所有元素 |
| `:` | 读取所有元素 |

```
import torch

# 创建一个形状为 [4, 32, 32, 3] 的 4D 随机张量
x = torch.randn(4, 32, 32, 3)

# dim 0: 4 (不变)
# dim 1: (28 - 0) / 2 = 14
# dim 2:  (28 - 0) / 2 = 14
# dim 3: 3 (不变)
s = x[:,0:28:2,0:28:2,:]
print(s.shape) # torch.Size([4, 14, 14, 3])
```

当张量维度很高（如 5D, 6D）时，写一堆冒号 `[:, :, :, :]` 既繁琐又易错。`...` 代表 “中间所有未指定的维度”。
| 切片方式 | 意义 |
| :--- | :--- |
| `a, ..., b` | `a` 维度对齐到最左边，`b` 维度对齐到最右边，中间的维度全部读取，其他维度按 `a/b` 的方式读取 |
| `a, ...` | `a` 维度对齐到最左边，`a` 维度后的所有维度全部读取，`a` 维度按 `a` 方式读取。这种情况等同于 `a` 索引/切片方式 |
| `..., b` | `b` 维度对齐到最右边，`b` 之前的所有维度全部读取，`b` 维度按 `b` 方式读取 |
| `...` | 读取张量所有数据 |

```
import torch

# 创建一个形状为 [4, 32, 32, 3] 的 4D 随机张量
x = torch.randn(4, 32, 32, 3)

s =  x[0:2,...,1:]
print(s.shape) # torch.Size([2, 32, 32, 2])

s = x[2:,...]
print(s.shape) # torch.Size([2, 32, 32, 3])
```
